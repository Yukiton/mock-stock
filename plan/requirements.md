# 模拟炒股后端API项目 - 需求文档

## 一、项目背景

构建一个模拟炒股后端API系统，支持用户进行股票模拟交易、持仓管理、资产估值、价格提醒等功能。系统需集成akshare库获取实时股票行情数据。akshare库代码在D:\playground\akshare。

---

## 二、核心功能需求

### 2.1 用户与资产管理

| 功能 | 描述 |
|------|------|
| 用户管理 | 支持用户注册、登录、信息查询 |
| 余额管理 | 管理用户账户现金余额 |
| 手动调整 | 管理员可手动调整用户余额和持仓（模拟充值、修正数据等） |

### 2.2 持仓管理

| 功能 | 描述 |
|------|------|
| 持仓查询 | 查询用户当前持有的股票及数量 |
| 持仓成本 | 记录每只股票的持仓成本价 |
| 手动调整 | 管理员可手动调整持仓数据 |

### 2.3 交易功能

| 功能 | 描述 |
|------|------|
| 买入股票 | 用户可购入股票，**成交价格可自定义**（不强制使用当前行情价） |
| 卖出股票 | 用户可卖出持仓股票，**成交价格可自定义** |
| 交易记录 | 完整记录所有买卖交易历史 |

### 2.4 资产估值

| 功能 | 描述 |
|------|------|
| 股票市值 | 调用akshare获取实时行情，计算持仓股票总市值 |
| 总资产 | 现金余额 + 股票市值 |
| 参考盈亏 | 根据持仓成本与当前价格计算盈亏情况 |

### 2.5 价格提醒系统

| 功能 | 描述 |
|------|------|
| 提醒设置 | 新增/修改/删除价格提醒配置 |
| 提醒触发 | 根据设定条件触发提醒 |

#### 提醒逻辑（抽象设计）

提醒逻辑设计为**抽象类**，支持多种量化策略实现：

- **阈值策略**: 价格突破设定上限/下限时触发
- **均线策略**: 价格突破MA5/MA10/MA20等均线时触发
- **MACD策略**: 基于MACD指标触发
- **RSI策略**: RSI超买/超卖时触发
- **自定义策略**: 可扩展的其他量化策略

#### 提醒接收器（抽象设计）

提醒接收器设计为**抽象类**，支持多种通知方式：

- **WebSocket**: 实时推送消息到前端
- **SMTP**: 发送邮件通知
- **Webhook**: 调用外部HTTP接口
- **MCP**: 通过MCP协议调用

### 2.6 定时任务

| 功能 | 描述 |
|------|------|
| Cron任务 | 支持cron表达式配置定时任务 |
| 价格查询 | 定时查询股票价格并记录 |
| 提醒检查 | 定时检查价格提醒条件是否满足 |

---

## 三、数据存储需求

### 3.1 数据库选型

- **SQLite**: 轻量级本地数据库，无需额外服务部署

### 3.2 数据表需求

| 表名 | 用途 |
|------|------|
| users | 用户信息、余额 |
| positions | 股票持仓、成本价 |
| transactions | 交易记录 |
| price_alerts | 价格提醒配置 |
| cron_jobs | 定时任务配置 |

---

## 四、行情数据需求

### 4.1 数据来源

- 使用项目内的 **akshare** 库获取股票行情
- 关键API:
  - `stock_bid_ask_em(symbol)`: 获取实时行情报价（含最新价、买卖五档等）
  - `stock_individual_info_em(symbol)`: 获取股票基本信息

### 4.2 行情数据用途

- 计算持仓市值
- 计算参考盈亏
- 触发价格提醒
- WebSocket实时推送

---

## 五、接口需求

### 5.1 用户接口

- 获取用户信息
- 获取/调整账户余额

### 5.2 持仓接口

- 获取持仓列表
- 获取单只股票持仓详情
- 手动调整持仓

### 5.3 交易接口

- 买入股票（可自定义价格）
- 卖出股票（可自定义价格）
- 查询交易历史

### 5.4 资产估值接口

- 获取股票持仓市值
- 获取总资产（现金+股票）
- 获取持仓盈亏情况

### 5.5 价格提醒接口

- 新增价格提醒
- 修改价格提醒
- 删除价格提醒
- 查询提醒列表

### 5.6 定时任务接口

- 新增定时任务
- 修改定时任务
- 删除定时任务
- 查询任务列表

### 5.7 实时行情接口

- 查询单只股票行情
- 批量查询行情
- WebSocket实时推送
